# keenetic-vpn-mihomo

VPN-клиент на базе [mihomo](https://github.com/MetaCubeX/mihomo)
(clash.meta) для роутеров **Keenetic** с Entware. Один скрипт ставит весь
стек, а через VPN ходят только те устройства, которые вы явно выбрали, —
остальная сеть работает напрямую. Принимает обычные happ-style
**VLESS-подписки**; сервер и подписку можно менять в любой момент.

## Что умеет

- **Per-device VPN**: в VPN ходят только выбранные устройства. Выбирать можно
  двумя способами (работают одновременно): мышкой в веб-админке Keenetic —
  через политику доступа «Mihomo», которую установщик создаёт сам, — или из
  консоли: `te-vpn add <mac|ip>`.
- **Подписки**: `te-vpn sub "<url>"` — вставил URL, mihomo сам тянет и обновляет
  список серверов.
- **Выбор сервера**: вручную (`te-vpn select`), автоматически по health-check
  (`te-vpn auto`) или мышкой в веб-интерфейсе.
- **Автофоллбек**: если mihomo упал — устройства автоматически идут напрямую
  (интернет не пропадает), monit тем временем перезапускает процесс.
- **Автозапуск**: init-скрипт при загрузке + monit-супервизия + NDM-хуки
  (переподключение WAN, пересборка файрвола).
- **Веб-интерфейс**: metacubexd на `http://<ip-роутера>:9090/ui` — статус,
  замеры задержек, переключение серверов, живой трафик.

## Как это работает

VPN-трафик отбирается по fwmark, меток две — по одной на каждый способ
выбора устройств:

1. Устройства из **политики доступа Keenetic** метит сама прошивка: у каждой
   политики есть свой connmark, `mihomo-route` находит его по имени политики
   (`ndmc -c "show ip policy"`). Устройства из **`devices.list`** метит наш
   chain в `mangle/PREROUTING` (по MAC или IP) fwmark'ом `0x6d69`.
2. mihomo поднимает TUN `mihomo0`. Скрипт `mihomo-route` добавляет для обеих
   меток `ip rule … lookup 1000 priority 99` (раньше правил Keenetic на 100),
   а в таблице 1000 — `default dev mihomo0`.
3. Помеченный трафик попадает в TUN → mihomo заворачивает его в VLESS и
   отправляет через WAN. Остальные устройства ходят напрямую.
4. **Фоллбек**: умер mihomo → исчез TUN → таблица 1000 пуста → правила 99 не
   срабатывают и трафик проваливается в обычную маршрутизацию (у политики
   для этого есть свой маршрут через WAN — установщик делает `permit`).
   Чинить руками ничего не нужно.

NDM-хуки держат это состояние живым: `netfilter.d` восстанавливает маркировку
после пересборки файрвола прошивкой, `wan.d` — правила роутинга после
переподключения WAN.

Все параметры (fwmark, имя политики, таблица, приоритет, имя TUN, путь к
списку устройств) собраны в `/opt/etc/mihomo/vpn.conf`.

## Установка

Нужен роутер с Entware и SSH-доступом (по умолчанию `root@192.168.1.1:222`,
переопределяется `ROUTER=… PORT=…`). Скрипт сам ставит зависимости через
opkg (curl, iptables, monit), определяет архитектуру CPU и качает нужный
бинарник mihomo. Повторный запуск безопасен и работает как обновление —
живые `config.yaml` / `vpn.conf` / `devices.list` не затираются (новые
версии лягут рядом как `*.new`).

```sh
sh scripts/install.sh
ssh root@192.168.1.1 -p 222 'te-vpn sub "https://your-subscription-url"'
ssh root@192.168.1.1 -p 222 'te-vpn auto'    # или: te-vpn select "🇳🇱 Server-1"
```

Дальше остаётся выбрать устройства (см. следующий раздел).

Флаги: `POLICY_NAME=MyVPN` — своё имя политики доступа, `WAN_IF=…` — имя
WAN-подключения, если оно не стандартное `ISP` (нужно политике для фоллбека),
`NO_POLICY=1` — не создавать политику (управление только через `te-vpn add`),
`NO_MONIT=1` — без monit-супервизии, `NO_START=1` — только разложить файлы,
не запускать. Удаление: `sh scripts/uninstall.sh` (конфиг и подписка
сохраняются; `PURGE=1` — снести всё).

## Выбор устройств: кто ходит через VPN

Два способа, работают одновременно; всё остальное в сети ходит напрямую.

**Через веб-админку (политика доступа).** Установщик сам создаёт в прошивке
политику доступа **Mihomo**. Назначьте ей устройства — и они пойдут через
VPN:

1. Откройте админку → **«Мои сети и Wi-Fi» → «Список устройств»** и
   зарегистрируйте нужное устройство, если оно ещё не зарегистрировано.
2. В карточке устройства в поле **«Политика доступа в интернет»** выберите
   **Mihomo**. Готово — трафик устройства идёт через VPN.
3. Сами политики живут в **«Сетевые правила» → «Приоритеты подключений»** —
   там политику Mihomo видно, но менять в ней ничего не нужно.

Убрать устройство из VPN — вернуть ему политику по умолчанию.

**Через консоль (`devices.list`).** Для устройств, которых нет в админке,
или если политика не нужна:

```sh
te-vpn add aa:bb:cc:dd:ee:ff laptop   # по MAC
te-vpn add 192.168.1.50 tv           # по IP или подсети (192.168.1.64/28)
te-vpn remove aa:bb:cc:dd:ee:ff
te-vpn devices
```

Если политика уже была создана раньше (или создана вручную в админке) —
просто впишите её имя в `POLICY_NAME=` в `/opt/etc/mihomo/vpn.conf`, скрипт
найдёт её метку сам.

## Повседневное использование

```sh
te-vpn status                 # процесс, TUN, устройства, роутинг, сервер, подписка
te-vpn add <mac|ip> [коммент] # добавить устройство в VPN
te-vpn remove <mac|ip>        # убрать устройство
te-vpn devices                # список устройств
te-vpn sub "https://..."      # сменить подписку
te-vpn servers                # список серверов
te-vpn select "<name>"        # переключить сервер вручную
te-vpn auto                   # первый живой сервер по health-check
te-vpn restart
te-vpn log [N]
```

## Структура репозитория

Дерево `router/` зеркалит `/opt` на роутере — файл лежит в репо там же, куда
он попадёт при деплое:

| В репо (`router/…`) | На роутере (`/opt/…`) | Что это |
|---|---|---|
| `etc/mihomo/config.yaml` | `etc/mihomo/config.yaml` | конфиг mihomo: подписка, группы, TUN, DNS |
| `etc/mihomo/vpn.conf` | `etc/mihomo/vpn.conf` | параметры маркировки/роутинга (fwmark, имя политики, таблица, TUN) |
| `etc/mihomo/devices.list` | `etc/mihomo/devices.list` | устройства, добавленные через `te-vpn add` |
| `bin/te-vpn` | `bin/te-vpn` | CLI управления (start/status/add/sub/select/…) |
| `bin/mihomo-route` | `bin/mihomo-route` | маркировка + policy-роутинг (add/del/sync/status) |
| `etc/init.d/S06mihomo` | `etc/init.d/S06mihomo` | init/автозапуск |
| `etc/monit.d/mihomo.conf` | `etc/monit.d/mihomo.conf` | monit-супервизия |
| `etc/ndm/wan.d/10-mihomo.sh` | `etc/ndm/wan.d/10-mihomo.sh` | восстановление роутинга при WAN up |
| `etc/ndm/netfilter.d/50-mihomo.sh` | `etc/ndm/netfilter.d/50-mihomo.sh` | восстановление маркировки после пересборки файрвола |

Бинарник mihomo (`/opt/sbin/mihomo`) и файлы веб-UI (`/opt/share/mihomo/ui`)
в репо не хранятся — install-скрипт скачивает их на роутер с GitHub-релизов.

`scripts/`:

- `install.sh` — полная установка/обновление (идемпотентно, см. выше)
- `uninstall.sh` — удаление

## Требования

- Роутер Keenetic (или иной) с Entware в `/opt` и SSH-доступом из LAN
- Архитектуры: aarch64 / armv7 / armv5 / mips(el) / x86_64 (определяется
  автоматически; версия mihomo задаётся в `scripts/install.sh`)
- Проверено на Keenetic NC-2312 (MT7981B, aarch64, kernel 4.9-ndm-5)
